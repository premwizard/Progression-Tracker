# Skill: Core Project Rules
**This is the most important skill. Always follow these rules.**

1. Never duplicate code.
2. Always write production-quality code.
3. Always separate business logic.
4. Always use reusable components.
5. Always write documentation.
6. Always update tests.
7. Always consider scalability.
8. Always follow clean architecture.
9. Always follow SOLID principles.
10. Always optimize performance.
11. Always prioritize accessibility.
12. Never break existing APIs.
13. Never hardcode secrets.
14. Always update related documentation after implementation.
